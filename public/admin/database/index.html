Cara kerja: Timer backup berjalan di dalam proses Node.js COCKPIT. Backup akan berjalan otomatis selama server aktif. Jika server restart, timer akan dimulai kembali dan backup berikutnya akan dihitung dari backup terakhir.

Lokasi penyimpanan: File backup disimpan di ~/cockpit_backups/ (home directory server) agar tidak hilang saat deploy ulang. Bisa dikustomisasi via env var BACKUP_DIR.
⚠ Peringatan: Restore akan mengeksekusi perintah SQL dari file yang diunggah langsung ke database. Pastikan file backup valid sebelum melanjutkan. Proses ini tidak bisa dibatalkan.
