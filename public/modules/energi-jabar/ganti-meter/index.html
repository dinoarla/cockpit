Sumber Data

Data berasal dari sistem pencatatan penggantian kWh meter PLN Distribusi Jawa Barat. File mentah berformat XLS, satu file per hari per wilayah (rekap harian). Setiap baris merepresentasikan satu transaksi penggantian meter.

Kolom Kunci yang Digunakan

  • UNITAP (col2) — Kode UP3, mis. 53BDG = Bandung
  • JENIS_MK (col17) — J/HJ/JM = Pascabayar; JK/HJK = Prabayar
  • ALASAN_GANTI_METER (col31) — Alasan teknis penggantian
  • MERK_METER_BARU (col35) — Merk meter pengganti
  • THBUAT_METER_BARU (col38) — Tahun produksi meter baru
  • MERK_METER_LAMA (col40) — Merk meter yang diganti
  • THBUAT_METER_LAMA (col43) — Tahun produksi meter lama (sejak 1950-an)

Grup Alasan Penggantian

  • Program — Program Migrasi AMI, Pemeliharaan Meter, Upgrade KRN
  • Rusak Display — LCD blank, error, Periksa, Inscek
  • Rusak Fisik — Cover pecah, terbakar, bencana alam, serangga
  • Gangguan Token — KCT tidak masuk, kredit reset/nol/tidak berkurang
  • Gangguan Teknis — Firmware, relay, keypad, port komunikasi
  • Lainnya — Alasan di luar kategori di atas

Cara Mengimpor Data

  • Pastikan folder ~/Downloads/Ganti Meter/ berisi subfolder tahun 2024, 2025, 2026
  • Jalankan: python3 drizzle/import_ganti_meter.py
  • Jalankan drizzle/register_ganti_meter.sql di phpMyAdmin untuk mendaftarkan modul
  • Restart Cockpit: node dist/index.js

Catatan Metodologi

  • Data mencakup penggantian prabayar (JK, HJK) dan pascabayar (J, HJ, JM)
  • Tanggal rekap diambil dari nama file; file multi-hari (mis. 01_03Mar) dihitung pada hari pertama
  • Tahun buat meter lama difilter 1950–2025 untuk analisis umur
  • UP3 yang tidak dikenal (bukan 17 UP3 Jabar) dilewati
  • Tahun 2024 dimulai bulan Maret (data sebelumnya tidak tersedia)
