📋 Tentang Modul Ini

Sumber Data

Data berasal dari sistem pembacaan meter (Cater/SWACAM) PLN Distribusi Jawa Barat. File mentah berformat Excel (.xlsx), satu file per UP3 per bulan. Setiap baris merepresentasikan satu pembacaan meter untuk satu pelanggan pascabayar.

Kolom Data Utama

  • UP3 — Kode unit pelaksana pelayanan pelanggan (mis. 53BDG = Bandung)
  • ULP — Unit layanan pelanggan (sub-unit di bawah UP3)
  • IDPEL — ID pelanggan (12 digit)
  • TARIF — Golongan tarif (R1, R2, R3, S1, B1, B2, P1, P3, I1, I2, L)
  • DAYA — Daya kontrak dalam VA
  • TANGGAL_PEMBACAAN — Tanggal meter dibaca petugas
  • KODE_PESAN — Status pembacaan (Z-NORMAL = berhasil normal)
  • PEMAKAIAN_KWH — Pemakaian listrik bulan berjalan (kWh)

Kode Pesan Umum

  • Z - NORMAL — Pembacaan normal, kWh terverifikasi
  • F - RUMAH KOSONG — Tidak ada aktivitas penghuni
  • E - RUMAH TUTUP — Pintu/akses terkunci
  • N - KWH LEBIH TAGIH — Anomali pemakaian tinggi
  • 2 - GANTI KWH METER — Pergantian meter, stan direset
  • A - KWH METER MACET — Meter tidak berfungsi normal
  • 1 - KWH METER TINGGI — Melebihi batas wajar

Cara Mengimpor Data

  • Pastikan folder ~/Downloads/DATA BACA METER JABAR 2026 berisi subfolder bulan (202602, 202604, dst.)
  • Jalankan: python3 drizzle/aggregate_baca_meter.py > /tmp/baca_meter_import.sql
  • Import /tmp/baca_meter_import.sql ke phpMyAdmin (database u164655286_cockpit)
  • Jalankan drizzle/register_baca_meter.sql untuk membuat tabel dan mendaftarkan modul
  • Restart Cockpit: node dist/index.js

Catatan Metodologi

  • Data hanya mencakup pelanggan pascabayar; pelanggan prabayar (token) tidak termasuk
  • Nilai PEMAKAIAN_KWH negatif dikecualikan dari agregasi (terjadi saat pergantian meter)
  • Juni 2026: file unit dari subfolder V2 (data revisi)
  • Bulan Maret 2026 tidak tersedia dalam dataset ini
  • Kode Sumedang berubah: SDM (Feb) → SMD (Apr+), digabung sebagai Sumedang
